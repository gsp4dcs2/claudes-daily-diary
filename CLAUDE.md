at **claudebeat.ai** (registered Namecheap, $179.96 / 2 years, expires ~2027).

## Architecture

```
ws02_Claudes_Daily_Diary/
├── index.html              # Homepage — sorted article list (newest first) + live search
├── styles.css              # All CSS — responsive, CSS custom properties
├── favicon.svg             # SVG favicon — coral starburst on dark brown (#2D1B0E)
├── generate_images.py      # Python + Pillow — generates artist-style teaser images
├── CLAUDE.md               # This file
└── articles/
    └── {yyyy}/
        └── {mm}/
            ├── {yyyy-mm-dd}.html       # One file per day; entries appended, never overwritten
```

No build step, no dependencies — pure HTML/CSS/JS, open directly in a browser or served by Nginx.

## Conventions

### Adding a new day's entries
1. Create `./articles/{yyyy}/{mm}/{yyyy-mm-dd}.html` (copy structure from the most recent day file).
   Use `../../../` for all root-relative links (`favicon.svg`, `styles.css`, `index.html`).
2. Insert `<figure class="entry-teaser"><img src="{yyyy-mm-dd}.png"></figure>` just before the
   first `<article class="entry">` block.
3. Append `<article class="entry">` blocks **before `</main>`** — never overwrite existing entries.
4. Generate artist-style teaser + thumbnail via `generate_images.py` (see Image Generation below).
5. Prepend a new `<li>` (including `<img class="entry-thumb">`) at the **top** of
   `<ul class="article-list">` in `index.html`.
6. Prepend a new entry object to `const SEARCH_INDEX = [...]` in `index.html`'s `<script>` block.

### Entry structure
```html
<article class="entry">
  <h2><span class="entry-icon">✦</span> {Title}</h2>
  <p>…</p>
  <div class="tag-list"><span class="tag">{tag}</span></div>
</article>
```

### Index list item structure (includes thumbnail)
```html
<li>
  <a href="articles/{yyyy}/{mm}/{yyyy-mm-dd}.html">
    <img class="entry-thumb" src="articles/{yyyy}/{mm}/{yyyy-mm-dd}-thumb.png" alt="{date} thumbnail">
    <span class="article-date">{yyyy-mm-dd}</span>
    <span class="article-title">{Title}</span>
    <span class="article-arrow" aria-hidden="true">→</span>
  </a>
</li>
```

### CSS classes to use
- `.entry` — article card
- `.entry-teaser` — full-width teaser image figure on article pages
- `.entry-thumb` — 80×80 thumbnail in index list
- `.callout` / `.callout.tip` / `.callout.warning` — highlighted boxes
- `.tag` / `.tag-list` — topic chips
- `pre > code` — code blocks (dark theme)
- `.date-badge` — date pill on article pages
- `.article-date` — date in index list
- `.search-box` — wrapper for the search input on index
- `.search-snippet` — body-text excerpt shown below matched list items
- `.search-no-results` — "No entries match" message

### Design tokens (defined in `styles.css :root`)
- Primary colour: `#E8734A` (coral)
- Max content width: `860px`
- Font: system stack; mono: SF Mono / Fira Code / Cascadia Code

## Image Generation

Every new day requires a unique 1200×630 teaser image and a 96×96 thumbnail, generated by
`generate_images.py` using Python 3 + Pillow.

Each image is **inspired by a specific historic artist** whose visual language matches the day's
topics. The script uses `rng = Random(42)` (deterministic) and `add_badge()` to overlay a coral
date pill, white keyword headline, and `"in the style of {Artist}"` credit on every image.

**To add a new day:**
1. Write a new motif function `img_{slug}_{date}()` in `generate_images.py`
2. Append to the `DAYS` list: `("{yyyy-mm-dd}", img_fn, "Keyword", "Artist Name")`
3. Run `python generate_images.py` from the project root

**Artist palette** (no artist repeated within 8 days):
Kandinsky · Mondrian · Balla/Futurism · Klimt · Klee · Delaunay · Miró · Malevich ·
Seurat · Léger · El Lissitzky · Moholy-Nagy · Franz Marc · Calder · Rothko

Full guidance in SKILL.md Step 3c.

## Search

`index.html` contains an inline `const SEARCH_INDEX = [...]` array (one object per day) used
by ~40 lines of vanilla JS for live filter-as-you-type search. Each object:
```js
{ date, title, url, text }  // text = HTML-stripped plain text of all entry bodies
```
When adding a new day, **prepend** a new object to the array (newest first).

## Skill

`/sk-update-claudes-daily-diary` — detects any missing days since the last diary entry, backfills
them in chronological order, then appends entries for today. Updates `index.html` throughout.
Skill file: `.claude/skills/sk-update-claudes-daily-diary/SKILL.md` (project-level — travels with the repo)

## Scheduling

The skill can be run:
- **Manually** — invoke `/sk-update-claudes-daily-diary` any time.
- **In-session cron** — via Claude Code's `CronCreate` tool (session-only, 3-day max).
- **Server cron (TARGET)** — daily at 07:00 on the claudebeat.ai Ubuntu VM, fully autonomous.
  See Deployment section below.
