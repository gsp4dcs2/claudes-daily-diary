Enterprise Analytics API & VS Code Plan Preview Improvements

🧭 Enterprise Analytics API — Programmatic Usage & Cost Visibility

Anthropic has launched the Enterprise Analytics API, giving workspace administrators programmatic access to daily usage and engagement data across Claude, Claude Code, and Cowork. Each endpoint returns a per-organisation, per-day snapshot covering request counts, token consumption, active user counts, and tool-call breakdowns. The API is accessible to Enterprise plan owners via standard REST calls.

What you can build with it

  • Cost attribution dashboards — pipe data into Snowflake, BigQuery, or Tableau to track spend by team, project, or user
  • Budget alerts — trigger Slack or PagerDuty notifications when daily token spend crosses a threshold
  • Usage anomaly detection — flag unusual spikes that may indicate runaway agent loops or misuse
  • ROI reporting — correlate AI usage data with engineering output metrics for executive reporting

Previously, spend visibility required manual Console exports — a workflow that did not scale for organisations managing hundreds of developer seats. The Analytics API is the most-requested Enterprise feature since SSO, according to Anthropic's customer success team.

🧭 VS Code Plan Preview — Live Iteration Feedback in Claude Code

A targeted improvement has landed in the Claude Code VS Code extension: the plan preview panel now auto-updates as Claude iterates on its execution plan, rather than displaying a static snapshot taken at task start. Two companion changes ship alongside it:

  • The comment overlay on the plan is suppressed until the plan is stable enough to review, eliminating the visual noise of watching Claude edit its own reasoning mid-stream
  • The preview pane stays open when a developer rejects a plan, allowing Claude to revise in-place rather than opening a new panel — preserving context and reducing click overhead

Together these changes tighten the review loop for developers who use the approval workflow before letting Claude write to disk. The previous behaviour — where rejecting a plan caused a fresh blank panel to open — was a frequently cited friction point in Claude Code user interviews.
