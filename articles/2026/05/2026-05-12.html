🧭 Google's Anthropic Stake Becomes a Material Investment — What $40B Buys Alphabet Investors

As Anthropic's implied valuation has climbed past $1.4 trillion, Google's cumulative investment of up to $40 billion in the company is crossing the threshold that requires Alphabet to recognise it as a material equity investment on its balance sheet — a disclosure that Motley Fool flagged on May 11 as "great news for Alphabet investors." At current implied valuations, Google's Anthropic stake represents a paper gain that is material relative to Alphabet's overall market capitalisation. This changes the reporting narrative: Anthropic's growth is now directly visible in Alphabet's quarterly filings rather than being disclosed only in footnotes.

The investor implications

  • Alphabet balance sheet lift — if Anthropic goes public at or near its current implied valuation, Alphabet's stake becomes a marked-to-market publicly-traded position, potentially adding hundreds of billions in book value.
  • Revenue upside — Anthropic's multi-year commitment to Google Cloud (reported at up to $200 billion over five years in some estimates) means Anthropic's growth directly benefits Google Cloud revenue, which had been a competitive weak point for Alphabet against AWS and Azure.
  • Strategic AI positioning — Alphabet can point to Anthropic's Claude as evidence of serious frontier AI investment even as its own Gemini models compete in the consumer space. The two are not direct substitutes from Alphabet's perspective.
What this means if you build on Claude via Vertex AI

The deeper the commercial ties between Google and Anthropic, the more Vertex AI becomes a first-class deployment path for Claude — not just a distribution channel. If your team is weighing Bedrock vs. Vertex AI for Claude deployments, the Google-Anthropic relationship suggests Vertex will receive API parity and new model access at least as fast as Bedrock, with the added benefit of tight GCP infrastructure integration (VPC Service Controls, BigQuery connectors, Vertex AI Pipelines). Neither is clearly superior for all use cases, but the commercial alignment favours Vertex for Google-heavy infrastructure shops.

🧭 Claude Mobile Adds Fully Interactive Apps — Live Charts, Diagrams, and Shareable Assets in Conversations

Anthropic's mobile app update rolling out this week enables Claude to connect to and render fully interactive apps within conversations — users can pull up live charts, sketch diagrams, and build shareable assets that are visually rendered directly in the chat interface rather than described in text or linked externally. This brings the mobile experience closer to the Claude.ai web app's artifact canvas functionality, closing a gap that had made mobile a second-class citizen for visual and data-exploration workflows.

How interactive apps work in mobile

When Claude generates an artifact (a chart, a table, an interactive diagram, a mini-application), the mobile client now renders it inline with native touch controls. Users can tap to interact with the artifact, modify it through follow-up messages, and share it as a standalone link. The feature works with the full range of Claude-generated artifact types: React components, Observable notebooks, SVG diagrams, and simple HTML applications.

Claude Code v2.1.139

Claude Code v2.1.139 shipped alongside the mobile update with improvements to model picking (smarter heuristics for which model to select for a given task complexity), stronger permission handling for tool calls in non-interactive mode, and improved OAuth login stability. The release also fixes a regression introduced in v2.1.138 where VS Code extension activation was intermittently failing on macOS after a sleep/wake cycle.

# Upgrade to get the macOS VS Code fix:
npm install -g @anthropic-ai/claude-code@latest

# Verify version:
claude --version
# Should show 2.1.139 or later
